If you are developer of some application and you would like too access graph storage of other users, you can define application access here. Data will be accessible using OAuth v1 protocol.
Client application setting
Key *

Unique identifier of your application. Should be short and simple.

Error message
Secret *

It is your "application password". Use Generate button.

Error message
Application name *

Name of your application which will be shown to end user.

Error message
Application url *

Url, where your application is accessible. Token callbacks will be sent there (if not specified otherwise in OAuth message)
