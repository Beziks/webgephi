Authentication and Authorization documentation

Authentication

In server application, we have 2 placed to authenticate: GUI app (on /index.jsp) and REST resources (on /rest).

GUI app

GUI application is used only for manage users and their consumer applications (and to authorize access to comsumer apps). Authentication id done using form. User have the same rights as the user logged to REST resources using BASIC auth method.

REST resources

If you want to access restricted REST resources, you have 2 possibilities to authenticate: BASIC and OAuth v1.

Method is selected depending on request header. OAuth is preferred. If no method is set, response with BASIC proposal is returned (so user can simply access resources using web browser).

Roles

User

Regular user if he logs in using BASIC or FORM. Can do anything with his data.

Permissions: 'USER', 'PROFILE_READ', 'PROFILE_WRITE', 'GRAPHS_READ', 'GRAPHS_WRITE'

Admin

Can do anything, permissions are the same as 'User' + 'ADMIN' role

Consumer application

User can give permission to access his account to some third party 'consumer' application (using OAuth protocol). This application, when access his resources later, has role 'Consumer application'. All permissions have to be requested during request token authorization and user sees which permissions consumer app requests.

Permissions: 'CLIENT_APP' and can request: 'PROFILE_READ', 'GRAPHS_READ', 'GRAPHS_WRITE'

Consumer applications, OAuth

If you want to access data of other users using OAuth protocol, you have to register your application first.

After that, you have to implement OAuth client in your application. Workflow is:

  1. Get request token from webgephi server (with requested scopes defined)
  2. Redirect user to webgephi server, where he authorizes request token (or not). After that, he is redirected back to your application with all necessary informations in url params.
  3. Exchange request token for access token
  4. Access REST resources using request token

If your application is written in java, you can use webgephi-client library. Or at least, you can inspire how to implement your own client. You can see how to use this library in webgephi-testclientapp project.
